def categorize_income(income):
    try:
        if 0 <= income <= 30000:
            return 'E'
        elif 30001 <= income <= 50000:
            return 'D'
        elif 50001 <= income <= 200000:
            return 'C'
        elif 200001 <= income <= 1000000:
            return 'B'
        elif income >= 1000001:
            return 'A'
    except:
        pass

def categorize_purpose(row):
    try:
        if 'автом' in row:
            return 'операции с автомобилем'
        elif 'жил' in row or 'недвиж' in row:
            return 'операции с недвижимостью'
        elif 'свад' in row:
            return 'проведение свадьбы'
        elif 'образов' in row:
            return 'получение образования'
    except:
        return 'нет категории'

У нас есть необходимость понять - является ли в данном случае наличие детей фактором влияющим на возврат задолженности. Применим для анализа независимости факта беспроблемного возврата задолженности от наличия детей китерий $\chi^2$. В этом случае мы рассмотрим гипотезы

$H_0$ = {возврат кредита не зависит от наличия детей}

$H_1$ = {возврат кредита зависит от наличия детей}

Возьмем стандартное для практических случаев пороговое значение $\alpha=0.05$

3.5 Приведите возможные причины появления пропусков в исходных данных.¶

Ответ: порпуски в исходных данных могут появиться из-за

  • человеческого фактора при вводе данных с физических источников (оператор пропустил данные при вводе с бумаг)
  • технических ошибок (например разрыв связи и некорректная обработка разрыва в процессе парсинга сайта с данными)
  • нежелание респондентов отвечать на вопросы из опроса или заполнять необязательные поля формы такие как электронная почта для рассылки рекламы
  • невозможность провести измерения в определенный день (например рулетка потерялась и мы не можем измерить рост человека в пятницу)
  • проблема с чтением данных с жесткого диска (битые сектора и тд)
  • проблемы с чтением данных из облаков при генерации сводных таблиц из разных источников (просроченная оплата облака, санкции и тд)
  • смена политики ввода данных (например, до сентября не было в таблице поля "зарплата", а в сентябре появилось. Следоватеьлно у всех объектов до сентября не будет занесено информации про зарплату)

3.6 Объясните, почему заполнить пропуски медианным значением — лучшее решение для количественных переменных.¶

Ответ: есть 2 меры среднего для количественных переменных.это среднее арифметическое и медиана. Медиана гораздо лучше отражает суть если данные распределены не по ноормальному закону. В случае когда данные подчиняются гауссовому распределению мы можем использовать среднее. Однако никто не может нам гарантировать что все наши распределения будут нормальными. Поэтому лучше взять медиану.
